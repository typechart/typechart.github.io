TypeChart is
Readable
Progressive
Readable
We try to make the syntax is readable.
What the syntax is readable means that the syntax can be easily mapped to elements in the chart.
Code
%%% type pie
Class A 10
Class B 20
Class C 30
Progressive
We means progressive writing and progressive learning.
You can write the charts in progressive manner.
 
➀ Jot down the key info.
%%% type pie
[Class A $n:10]
[Class B $n:20]
[Class C $n:30]
➁ Show % in slices also.
%%% type pie
%%% $pie.show:percent
[Class A $n:10]
[Class B $n:20]
[Class C $n:30]
➂ Break the text in slices into 2 lines.
%%% type pie
%%% $pie.show:percent
[Class A //# $n:10]
[Class B //# $n:20]
[Class C //# $n:30]
No Positioning
No Positioning! No xy!
Calculation of coordinations or dimensions is a hassle.
We try to make the syntax without positioning or dimensions as much as possible.
Code
%%% type ghostleg
%%% $k:3
a[Ashley] === 1[Apple]
b[Beric] === 2[Banana]
c[Carol] === 3[Cherry]
d[Dan] === 4[Dragon//Fruit]
